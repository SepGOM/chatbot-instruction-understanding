# 연구 계획서

## 1. 문제 정의

### 1.1 과제

- **Instruct Model의 이해 (Comprehension of Instruct Model)**

### 1.2 문제 해결 방법

어떠한 입력이 들어왔을 때 모델은 **요약, 분류, 질의응답** 중 사용자의 지시에 따라 수행을 한다.

- **요약**: 민원인과 상담원의 대화 내용과 같은 텍스트에서 중요한 정보들을 요약해준다.
- **분류**: 텍스트를 통하여 민원의 결과가 만족스러웠는지, 상담 원인이 무엇인지, 민원이 어떠한 민원인지 분류해준다.
- **질의응답**: 민원인의 질문에 답을 해준다.

### 1.3 과제의 중요성

위의 기능들이 잘 수행된다면, 이를 바탕으로 분야별 관련 문서를 검색하거나 참고하여 정확한 답변을 수행할 수 있다.

또한 세부적으로 추가 기능이나 기술을 도입하면 다음과 같은 활용이 가능하다:

- 상담원과의 전화를 통해 **실시간으로 추론**하여 전화 내용을 요약함으로써, 대화 내용을 잊더라도 다시 확인할 수 있다.
- **분류 기능**을 활용하여 상담원 답변의 질을 판단하거나, 민원인이 챗봇을 통해 본인의 민원을 어디에 접수해야 하는지 정확히 알 수 있어 흔히 겪는 전화 뺑뺑이를 피할 수 있다.
- 사용자는 **챗봇**을 통하여 간단한 정보들을 쉽게 해결할 수 있다.
